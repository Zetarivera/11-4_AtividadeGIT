GIT BRANCH

No Git, um "branch" (ramo) é uma versão independente do seu código. Imagine que você tem uma árvore, o tronco principal é o código principal (geralmente chamado de "main" ou "master") e os "branches" são como galhos que saem desse tronco.

Como funciona?

Branches no Git são versões paralelas do código, permitindo desenvolvimento isolado, colaboração e testes sem afetar o código principal. Comandos como "git branch", "git checkout" e "git merge" gerenciam a criação, troca e combinação de branches.


Para quê serve?

Um "branch" serve para isolar mudanças no código, permitindo desenvolver novas funcionalidades, corrigir bugs ou experimentar ideias sem afetar a versão principal do projeto.


Como criar e gerenciar branches?

Para criar um "branch" no Git, use git branch <'nome-do-branch>. Para mudar para ele, use git checkout <'nome-do-branch>. Para combinar um "branch" com outro, use git merge <'nome-do-branch>. Para excluir um "branch", use git branch -d <'nome-do-branch>.
